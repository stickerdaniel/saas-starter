**False positives caught by Phase 2 (technical verification):**

- `passkeyQueries.ts` public queries — actually protected by Convex component isolation
- `scrollbar-thin` in `@layer components` — actually correct for multi-property CSS patterns in Tailwind v4

**Fixes adjusted/dropped by Phase 3 (UX verification):**

- `#24` Noscript dark mode — **DROPPED**: Dark mode requires JS (ModeWatcher). Noscript users always see light mode, so `dark:` variants never apply.
- `#10` Icon-only buttons ARIA — **ADJUSTED**: Phone/Video/Info buttons are non-functional decorative placeholders (no click handlers). Adding ARIA labels to dead buttons misleads screen reader users. Fix: either add `aria-hidden="true"` + `tabindex="-1"` to hide them, or remove the buttons entirely.
- `#39` Avatar shuffle randomness — **ADJUSTED**: The re-shuffle on dependency change is actually the desired behavior (show different team member combinations). The real issue is only that `$t()` in the derived causes unnecessary re-shuffles on language change. Fix: memoize shuffle based on `adminUsers` identity only, not `$t()`.
- `#19` Passkey deletion confirmation — **ADJUSTED**: Passkeys can be re-registered trivially (just tap the fingerprint sensor again). A confirmation dialog adds friction to a low-risk, easily-reversible action. Other platforms (GitHub, Apple) do confirm, but they also can't re-register as easily. Fix: keep the quick-delete but add an undo toast (like email recipient deletion) instead of a blocking dialog.

## Architecture Highlights (Already Best-in-Class)

- **Zero Svelte 4 legacy** — complete Svelte 5 runes migration (0 instances of `export let`, `<slot>`, `$:`, `on:event`)
- **Consistent auth wrappers** — `authedQuery`/`adminQuery` pattern correctly applied on 95%+ of functions
- **Excellent deferred loading** — `requestIdleCallback` + dynamic imports for heavy modules (Rive, customer support, command menu)
- **All `{#each}` blocks keyed** — zero missing keys across entire codebase
- **Individual icon imports** — no barrel imports from `@lucide/svelte`
- **Proper event listener cleanup** — `$effect` return functions used consistently
- **HTML escaping in emails** — `escapeHtml()` applied to all user content in templates
- **Open redirect prevention** — `safeRedirectPath()` rejects absolute/protocol-relative URLs
- **Rate limiting on AI endpoints** — per-user + global limits on LLM cost
- **Field UI conventions followed** — `Field.Group > Field.Field > Field.Label + Input + Field.Error` everywhere, no explicit spacing on `Field.Group`
- **Font loading optimized** — self-hosted with `font-display: swap` and preloading
- **Proper timer cleanup** — `setTimeout`/`setInterval` cleared in all cleanup paths

## Prevention Strategy

### Automated: ESLint Rules

These recurring patterns can be caught at dev time with zero human effort.

| Findings                       | Rule                                                                 | Type                                   |
| ------------------------------ | -------------------------------------------------------------------- | -------------------------------------- |
| #20 `$app/stores`              | Ban import from `$app/stores` → suggest `$app/state`                 | `no-restricted-imports` (built-in)     |
| #13 Duplicate utils            | Ban import from `$lib/utils/utils` and `blocks/lib/utils`            | `no-restricted-imports` (built-in)     |
| #11 Missing `autocomplete`     | Warn on `<input type="email\|password">` without `autocomplete` attr | Custom Svelte AST rule (medium effort) |
| #38 Mixed `h-4 w-4` / `size-4` | Warn on paired `h-N w-N` where N matches                             | Custom regex rule (medium effort)      |
